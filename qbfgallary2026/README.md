# QBF Instances for Adversarial Games in GDL

The QBF instances answer the question: can a player get 100 points within n steps regardless of the actions of all the other players. The GDL descriptions of the games are available at: https://github.com/hharryyf/gdl2qbf-general/tree/main/SinglePlayer/GameDescriptions. The QBF instances are generated by the toolchain: GDL -> temproal extented GDL -> QASP -> QBF. The framework is formally introduced in our AAMAS 2024 paper: https://www.ifaamas.org/Proceedings/aamas2024/pdfs/p807.pdf

The GDL games considered include: 

1. breakthrough (2x5, 2x6, 3x4, 4x4, 3x5 for player o at depth 18, 3x5 for player x at depth 19) board. 
   -- instances are named as break-through-[width]x[height]-[depth]-[player].qdimacs
      SAT iff player can win within depth many steps. 

2. connect-3    (4x4, 5x5, and 6x6) board.
   -- instances are named as connect-3-[width]x[height]-[depth]-[player].qdimacs
      SAT iff player can win within depth many steps.

3. connect-4    (5x4, 5x5 at depth 22, 5x5) board.      <- connect-4-5x5 at full depth is hard 
   -- instances are named as connect-4-[width]x[height]-[depth]-[player].qdimacs
      SAT iff player can win within depth many steps.

4. dots-and-boxes    (2x2, 2x3, and 2x4) board.         <- dots-and-boxes-2x4 at full depth is hard
   -- instances are named as dots-and-boxes-[width]x[height]-[depth]-[player].qdimacs
      SAT iff player can win within depth many steps.

5. gttt-4x4-1-1  (elly, fatty, knobby, skinny, tippy)
   -- instances are named as gttt-4x4-1-1-[config]-[depth]-[player].qdimacs
      SAT iff player can win within depth many steps.

6. gttt-4x4-2-2  (elly, fatty, knobby, skinny, tippy) 
   -- instances are named as gttt-4x4-2-2-[config]-[depth]-[player].qdimacs
      SAT iff player can win within depth many steps.

7. 3-player game (whether the a player can force a win/draw, players are xplayer - oplayer -zplayer):
   (7.1) 3p-connect-3   (4x4, 5x5, 6x6) board 
      -- instances are named as connect-3-splayer-[width]x[height]-[depth]-[player].qdimacs
         SAT iff player can win or draw within depth many steps.
   (7.2) 3p-connect-4   (4x4, 5x4, 5x5 at depth 24, 5x5) board   <- 3p-connect-4-5x5 at full depth is hard
      -- instances are named as connect-4-splayer-[width]x[height]-[depth]-[player].qdimacs
         SAT iff player can win or draw within depth many steps.
   (7.3) 3p-tic-tac-tie (3x3, 4x4) board
      -- instances are named as tic-tac-toe-3player-[width]x[height]-[depth]-[player].qdimacs
         SAT iff player can win or draw within depth many steps.

GB/naive_encoding.zip contains instances generated with the baseline encoding (Def. 6 in the paper).

GD/dependency_based_encoding.zip contains instances generated with the dependency-based encoding (Def. 8 in the paper). These instances perform quantifier shifting based on the atom dependency graph of the GDL description. Correctness can be justified by the splitting theorem.

Generator is in the root directory: https://github.com/hharryyf/gdl2qbf-general/

Instances in the GD directory are friendly to both CEGAR-based solvers and QCDCL solvers after bloqqer preprocessing.
However, instances in GB are very unfriendly to CEGAR-based solvers but fine for QCDCL-based solvers.
